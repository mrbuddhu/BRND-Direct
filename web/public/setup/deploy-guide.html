Common Questions

Where exactly is the Download button in the builder?
Look at the top-right corner of the builder interface. It should say "Download", "Export", or show a ⬇ icon. If you don't see it, try looking in the menu or toolbar. The file downloaded will be a .zip file containing all the portal files.
Do I need to unzip the file before uploading?
No! Keep it as a ZIP. Cloudflare Pages accepts ZIP files directly. Do not unzip/extract it before uploading — just drag the .zip file directly onto the Cloudflare upload box.
What if I can't find "Upload assets" in Cloudflare?
Make sure you're clicking Pages first (not Workers), and then you should see two options: "Connect to Git" and "Upload assets". If you only see "Connect to Git", look for a direct URL option at the bottom of the page or try refreshing. You can also go directly to: dash.cloudflare.com → Workers & Pages → Create → Pages tab.
The upload says it succeeded but I get a blank page — what do I do?
Try visiting brnd-direct.pages.dev/buyer/ instead of just the root URL. The root index.html redirects to the buyer portal. If the buyer page also shows blank, check that the ZIP included all files (buyer/, css/, js/ folders). In Cloudflare Pages, click on the deployment and check "Files" to confirm the folder structure is correct.
The "UPDATE profiles SET role = 'admin'" returned 0 rows — why?
This means you're not logged in to Supabase, so auth.uid() returned null. You must be logged in to the portal first. Try: ① log in at brnd-direct.pages.dev/buyer/ ② then run the SQL immediately. If it still returns 0, go to Supabase → Authentication → Users, find your email, copy the UUID, and run: UPDATE profiles SET role = 'admin' WHERE id = 'PASTE-YOUR-UUID-HERE';
Can I update the site after deploying?
Yes! To update: ① Download the new version of the project as a ZIP ② In Cloudflare Pages → your project → Deployments → click "Create new deployment" ③ Upload the new ZIP. The site will update within seconds. Your domain stays the same.
