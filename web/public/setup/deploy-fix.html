Why does this happen?

🔑
GitHub not authorized
Cloudflare doesn't have permission to access your GitHub account or the specific repo.
📁
Repo is empty or private
The GitHub repo was created but no files were pushed to it yet, or it's set to private with no access granted.
🔗
Wrong repo selected
The Cloudflare Pages connection was pointed at a different or non-existent repository.

Re-authorize GitHub in Cloudflare
Do this first — fixes 90% of cases
1
Go to Cloudflare Dashboard → Workers & Pages

Open dash.cloudflare.com and click Workers & Pages in the left sidebar.

2
Find your Pages project and click into it

Click the project that showed the error (e.g. brnd-direct).

3
Go to Settings → Builds & Deployments

Inside the project, click the Settings tab, then click Builds & deployments in the left sub-menu.

Still broken? Delete the project and start fresh
Takes 2 minutes — sometimes the cleanest fix
1
Delete the broken Pages project in Cloudflare

Go to your Pages project → Settings → General → scroll to the very bottom → click Delete project.

2
Create a brand new Pages project

Click Create again → choose either Connect to Git (after fixing GitHub auth above) or Upload assets (simplest).

Deleting a Pages project does NOT delete your GitHub repo or any of your code. It only removes the Cloudflare deployment.
