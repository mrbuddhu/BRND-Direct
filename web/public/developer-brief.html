What is BRND Direct?

BRND Direct is a private, invitation-only B2B wholesale & dropship marketplace. It connects approved buyers (retailers, boutiques, e-commerce stores) with premium brand sellers (Chanel, Dior, Maison Margiela, etc.) for wholesale purchasing AND dropship order fulfillment. There is also a Shopify integration layer that allows buyers who have Shopify stores to auto-sync product catalogs and have orders fulfilled directly from BRND Direct into their Shopify storefront.

Buyer (Retailer)

Approved wholesale buyers who purchase products in bulk or use dropship mode. They browse the catalog, build orders, pay invoices on Net-30/60/90 terms, and track fulfillment. They may also connect their Shopify store to automate the entire dropship pipeline.

Seller (Brand)

Premium brands or their distributors who list products in the BRND Direct catalog. They receive and confirm orders, manage their product listings and pricing, pick-pack and ship, enter tracking info, and receive weekly/monthly payouts via Stripe Connect.

Admin (BRND Direct)

Internal BRND Direct staff who manage platform operations: approve buyers and sellers, oversee all orders, handle disputes, manage credit limits, configure platform settings, view all analytics, and manage payouts to sellers.

Two Fulfillment Models

Wholesale Model
Buyer browses catalog → adds to cart → confirms order → pays invoice (Net-30/60/90 or prepay) → seller packs → buyer books shipment via Freightos (LTL freight) or Shippo (parcel) → goods delivered to buyer's warehouse.

Dropship Model
Buyer connects their Shopify store → end customers place orders on buyer's Shopify → orders automatically route to BRND Direct → BRND Direct fulfills and ships directly to end customer → tracking pushed back to Shopify → daily invoice auto-generated for buyer.

Three Ways to Create an Order
1. Cart Checkout

Standard e-commerce flow. Browse products → add to cart → proceed to cart.html → confirm order. Generates order + invoice automatically.

2. AI PO Upload

Upload PDF/XLSX/CSV purchase order. AI parses the document → matches products to catalog → shows editable line items → generate order + invoice with one click.

3. Manual Order Builder

Search and select products manually via product picker grid. Set quantities, PO reference, shipping, payment terms → generate order + invoice.

Phase 1: Shopify Only (Current Focus)

The Dropship module is designed for Phase 1 Shopify integration. Phase 2 will add WooCommerce, Amazon, TikTok Shop, and Manual CSV Upload. The full UI is already built — only the backend/integration layer needs to be built.

Shopify Integration — 5-Step Setup Flow
Step 1 — Install Shopify App
Buyer visits the BRND Direct Shopify App listing on the Shopify App Store and clicks "Install". OAuth flow grants BRND Direct access to the merchant's store. Alternatively, buyer enters API credentials manually (Admin API Token + Secret).
Step 2 — API Credentials & Verification
Store the Shopify Admin API Token server-side (never in the browser). Required scopes: read_orders, write_orders, read_products, write_products, write_fulfillments. Validate by calling the Shopify API and confirming connection.
Step 3 — Catalog Sync to Shopify
Push the full BRND Direct product catalog to the merchant's Shopify store. Each product is created with wholesale price as cost, and Shopify retail price is set using the buyer's selected pricing rule (keystone ×2, 2.5×, 3×, custom markup, or MAP). Images, descriptions, SKUs, and barcodes are synced. Stock levels are updated in real time.
Step 4 — Automation Mode Selection
Full Auto: When a customer orders on the Shopify store, BRND Direct automatically ships the next business day. Requires active credit line (Net-30 or pre-pay). No manual confirmation needed. Auto-invoice generated daily.
Daily Confirm: Buyer receives email at 6 AM with a list of new orders. Must confirm by 11 AM EST. BRND Direct ships at 3 PM. Invoice generated at midnight. If buyer misses deadline, orders are held (configurable: hold / auto-confirm / cancel).
Step 5 — Go Live & Webhook Install
Install Shopify webhooks: orders/create, orders/updated, orders/cancelled. Begin listening for new orders. Confirm catalog sync completion (1,240 products, ~10 min). Enable tracking push webhook.
Order Import Flow (Shopify → BRND Direct)
  • Customer orders on buyer's Shopify store
  • Shopify fires orders/create webhook to BRND Direct Worker
  • Worker maps Shopify line items to BRND Direct SKUs
  • Creates order record in DB with source: 'shopify'
  • If Full Auto: marks as processing immediately
  • If Daily Confirm: adds to today's confirmation queue
  • At confirmation / auto-approval: begins fulfillment
Tracking Push (BRND Direct → Shopify)
  • BRND Direct ships order, gets tracking number
  • Calls Shopify Fulfillment API for that order
  • Creates fulfillment with carrier + tracking number
  • Shopify automatically emails end customer with tracking
  • Order marked as "Fulfilled" on Shopify
  • Target: <30 second delay after label creation
  • Success rate target: 99.9%+
Auto-Invoice Generation (Daily)

At midnight each day (or configurable time), BRND Direct generates a consolidated invoice for all dropship orders fulfilled that day. The invoice includes: wholesale cost per product, shipping fee per order, any applied credit, total due. Invoice is emailed as PDF + visible in dropship.html Auto-Invoices tab. If buyer has Net-30 credit, the due date is set 30 days out. If pre-pay, the card is charged automatically.

Credit / Net-Terms for Auto-Ship
Full Auto Mode Requirements:
  • Active Net-30, Net-60, or Net-90 credit line
  • OR pre-pay (card on file charged daily)
  • Good standing (no overdue invoices)
  • Manager approval required
Credit Line Management:
  • Credit limit stored in buyer_profiles.credit_limit
  • Used credit tracked in buyer_profiles.credit_used
  • Available = limit - used
  • Admin sets and adjusts credit limits

Row Level Security (RLS) — Already Defined

All 13 database tables have RLS enabled with pre-written policies in schema.sql. Key rules:

• Buyers can only see/modify their own orders, invoices, RFQs, and buyer_profile record.
• Sellers can see order items for their products, manage their own products/brands/payouts.
• Anyone can read active brands and products (public catalog).
• Admins have full access to all tables (via is_admin() helper function).
• Messages are only visible to conversation participants.
• Analytics events are write-only for the submitting user; admins can read all.
